三、JSON数据

①定义

JOSN是Javascript对象表示法（JavaScript Object Notation），轻量级的文本数据交换格式，独立于语言和平台，更快，更易解析

②JSON 与 JS 对象的关系

JSON 是 JS 对象的字符串表示法，它使用文本表示一个 JS 对象的信息，本质是一个字符串。

④常用类型

任何支持的类型都可以通过 JSON 来表示，例如字符串、数字、对象、数组等。

对象和数组是比较特殊且常用的两种类型。

对象：数据结构为 {key1：value1, key2：value2, …} 的键值对结构。key 为对象的属性，value 为对应的值。键名可以使用整数和字符串来表示，值的类型可以是任意类型。

数组：数据结构为 [“java”, “javascript”, “vb”, …] 的索引结构。数组是一种比较特殊的数据类型，数组元素也可以是对象。

六、get与POST请求方式的区别

没有前提，不使用任何规范，只考虑语法和理论上的HTTP协议。GET和 POST没有什么区别，只有名字不一样。

基于RFC规范：

理论上：GET和POST具有相同语法的，但是有不同的语义。get是用来获取数据 的，post是用来发送数据的，其他方面没有区别。

实现上：各种浏览器，就是这个规范的实现者。

常见的区别：

①GET的数据在URL是可见的，POST请求不显示在URL中。

②GET与POST相比，GET安全性较差，因为所发的数据是URL的一部分。

③GET对长度是有限制的，POST长度是无限的。

④GET请求的数据可以收藏为书签，post请求到的数据不可收藏为书签。

⑤GET请求后，按后退按钮、刷新按钮无影响，post数据会被重新提交。

⑥GET历史参数会被保留在浏览器里，psot不会保存在浏览器中。

⑦GET只允许ASCII.post没有编码限制，允许发二进制。

八、封装ajax方法

实际上自己封装Ajax请求，就是将原生Ajax的请求步骤抽象为一个函数，并单独生成一个js文件保存，当用到的时候直接引入这个文件的过程。

封装Ajax大致分成下列几个步骤：

(1)提供创建xhr对象的兼容性函数。

(2)提供发送请求的对外接口

(3)设计并约定对外接口的参数规格

(4)实现对外接口中参数处理

(5)实现对外接口中响应处理

(6)实现对外接口中发送处理

(7)设置命名空间，避免全局变量污染。

九、请求超时timeout与超时监听ontimeout（扩展）

描述：timeout属性等于一个整数，用来设置当请求发出后等待接收响应的时间。 ontimeout()方法则是当等待超时后，自动执行的回调方法。

语法：xhr.timeout = xxx; xhr.ontimeout=function(){ console.error(“The request for”+url地址+”timed out”); };

说明：timeout属性单位是毫秒，表示当请求发出后等待响应的时间。如果在设置的时间内没能收到后台响应的内 容，则认为请求超时(执行ontimeout)。

补充：如果该属性等于0，就表示没有时间限制。
